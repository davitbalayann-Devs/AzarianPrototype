In `Azarian Home.dc.html`, the Growth OS section (`#os` → `.gos2` → `.gos-net`) has an interactive tutorial: hovering the "SEO & search" node (`.gos-node[data-gos-seo]`, first left node) opens a centered "screen" (`.gos-screen[data-gos-screen]`, absolutely centered inside `.gos-net`, z-index 5, covering the logo core) that plays a scripted sequence (Figma 22578-8230 / -8339 / -8342):
1. screen fades+scales in;
2. a teal user-message bubble ("Pull the SEO performance for this client.") types in;
3. a "SEO Performance · last 30 days" results card appears and its 3 stat blocks (Organic Traffic 12.4k / Keywords Ranked 486 / Avg. Position 8.2) reveal one by one;
4. footer line "Need a performance report? Just ask."

THREE screens now share this engine: **SEO & search** (`[data-gos-seo]` → `[data-gos-screen]`), **CRM & Conversation** (`[data-gos-crm]` → `[data-gos-screen-crm]`, blue theme, Figma 22598-1243), and **Content** (`[data-gos-content]` → `[data-gos-screen-content]`, purple theme, Figma 22601-1547 — this node was renamed from "Paid media").

FOUR screens total; the fourth is **Intelligence** (`[data-gos-intel]` → `[data-gos-screen-intel]`, orange theme, Figma 22602-1927 — this node was renamed from "Web"). INTELLIGENCE variant (`_gosBuildScreen` branches on `screen.classList.contains('intel')`): a "Competitive snapshot" bar chart (`.gos-chart` → 4 `.gos-bar-col`, colours orange `#FF8642` / yellow `#FFCB13` / teal `#1BFED1` / blue `#5188BC`). On reveal the bars **rise from height 0** via a CSS `height` transition (`.gos-bar`, ~0.95s, staggered 100ms) inside a fixed-height `.gos-bar-wrap`; ONLY after the tallest finishes do the `.gos-bar-num` numbers fade in (`.show`) and count up (each `[data-countup]` above its bar, positioned `bottom:var(--h)`). No resize: the wrap is fixed height and the numbers are absolutely positioned. bar targets live in `data-h` (e.g. `80%`).

FIFTH screen: **Memory** (`[data-gos-mem]` → `[data-gos-screen-mem]`, teal/default theme, class `.gos-screen.mem`, Figma 22603-2095 — this node was renamed from "Automation"). It reuses the CONTENT-STYLE logic (skeleton → typed body) but with a different card: an "I Remembered" label + a `.gos-mem-meta` chips row (Alex Brown / date / Magnito Group / fireflies.ai) + title "Discussion with the Client" + a recalled-call paragraph that types in. Skeleton is theme-tinted via `--skel`/`--skel2` (teal for `.mem`, purple for `.content`).

CONTENT-STYLE variant is now detected by `isContent = !!screen.querySelector('[data-content-body]')` (NOT the `.content` class — so Memory triggers it too). Its card reveal BlurTexts every `.gos-screen-card [data-blur]` (works for Content's writer header and Memory's label+title). CONTENT variant details: a "Content Writer agent" card whose body first shows a **skeleton** (`.gos-skel` absolute-overlay shimmer bars) DURING the bot load (card reveals at `loadAt`, not `doneAt`), then after loading the skeleton fades and the body **types in** via `typeText(el, text, 1500)` — a vanilla reactbits **TextType** (rAF char reveal + blinking `.is-typing::after` caret, ~1.5s for the full ~450-char body). The body text lives in `data-content-body[data-text]` with paragraphs separated by `¶` (joined to `\n\n`, rendered `white-space:pre-wrap`). No-resize: on reset the full body text is measured and its height locked onto `.gos-body-slot { min-height }`, so swapping skeleton→typed text never changes the card height. Subtitle flips "Drafting…" → "Draft ready". `setupGosTutorial()` builds each via `_gosBuildScreen(node, screen)` (returns a controller: `play/hardClose/cancelClose/scheduleClose`), then wires each node's `pointerenter` to close the OTHER screens (mutual exclusion — they overlap in the centre) and open its own. `net` pointerenter cancels close, pointerleave schedules ~280ms close for all. To add another node tutorial: give the node `data-gos-X`, add a `.gos-screen` with `data-gos-screen-X`, and add the pair to the `specs` array. The CRM screen adds a `.gos-deal-head` (mini logo + title + "Proposal Sent" badge) and a `.gos-pipe` stage bar (4 segments; `.gos-pipe-seg.on::after` fills via CSS scaleX when the card gets `.in`); blue theme via `.gos-screen.crm` overrides; deal value `$48,000` uses count-up with `data-separator=","`.

Opens on the node's `pointerenter`; closes ~280ms after `pointerleave` of `.gos-net` (moving into the screen keeps it open since the screen is inside the net). Replays on every hover via `reset()` + timed `setTimeout` steps.

Sequence (Figma 22586-8759/-8851/-8806/-8807/-8810 / 22593-1377): screen opens → user message bubble + circular avatar (`.gos-screen-avatar`, `assets/hamlet-azarian.png`) type in → IRIS Lottie bot (`.gos-bot`, 40px) plays `assets/iris-logo-opening.json` once (`is-in`) then switches to looped `assets/iris-logo-loading.json` at **1.45× speed** (`loading`) for **LOAD** ms, with a Poppins 12px `#96A0AA` label **"Loading data"** + animated dots (`.gos-bot-label` / `.gos-bot-dots`) beside the icon → then rests on the opening end-frame (`is-rest`, label hides) while the results card + stat blocks reveal. Driven by `lottie-web` (`./lottie.min.js`).

IMPORTANT sizing rule (user requirement): the panel opens at its FINAL size and must NOT grow/shrink during playback. Achieved by (a) revealing sub-steps via opacity/transform only (elements always present), and (b) the two-phase BlurText — `buildBlur(el)` lays out the letter spans in their hidden start state at setup/reset so text width is reserved, `playBlur(el)` animates them. Space chars use a non-breaking space (` `, charCode 160) or inline-block spans collapse them. The screen is positioned `left:14%;right:14%` so it fills between — but never covers — the side node "buttons" (nodes sit at 9.66%/90.34%).

- Text uses a vanilla port of the reactbits **BlurText**, split into two methods: `buildBlur(el)` (build hidden letter spans, reserve layout) and `playBlur(el)` (animate blur+opacity+translateY per letter via Web Animations API, `fill:'both'`, staggered). Text cached to `el.dataset.text`; elements marked `[data-blur]`.
- Numbers use the existing `animateCount(el)` (`data-countup`/`data-decimals`/`data-duration`) — the same count-up used across the Home page. `animateCount`'s progress `t` is clamped `Math.max(0, …)` (harmless in real browsers; prevents a headless virtual-clock glitch).

Colors (Figma vars): orange stat labels `#ff8642`, teal `#1BFED1`, message bubble `#16C79A`, stat cards navy `rgba(6,59,109,.30)`. CSS lives with the other `#os` styles under the "Growth OS · SEO tutorial screen" comment.
